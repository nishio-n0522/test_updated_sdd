# 1. 基本概念

## スペック駆動開発とは

スペック（仕様書）を起点として、上流から下流へとドキュメントを導出し、最終的にコード（実装・テスト）に至る開発手法。
コードベースへの変更はすべて issue を通じて行い、autopilot skill によって自動実行される。

## ユーザーとAIの役割分担

| 役割 | ユーザー | AI |
|---|---|---|
| アイデアの壁打ち | 構想を伝える | 壁打ち相手として整理 |
| プロダクト仕様作成 | 確認・承認 | ドキュメント生成 |
| 実装仕様作成 | 確認・承認 | ドキュメント生成 |
| issue作成 | 既存PJ: 対話的に作成 / 新規PJ: 確認・承認 | issue分解・登録 |
| 実装 | PR確認 | autopilotで自動実行 |
| ドキュメント更新 | 確認 | docs/を更新 |
