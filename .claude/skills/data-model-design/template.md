## 分割の判断基準

- **エンティティ数が少ない場合**（〜5個）: 1つの層ファイルにまとめてもよい
- **エンティティ数が多い場合**（6個〜）: 関心事ごとに分割する
- **分割軸の例**:
  - 技術的な層: グラフモデル層 / プロセッサ定義層
  - ビジネスドメイン: ユーザー管理 / 注文管理 / 在庫管理
  - 機能設計書の機能グループに対応させると追跡しやすい
- **各ファイルは300行以下を推奨、400行を上限とする**
