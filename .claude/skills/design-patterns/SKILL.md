# デザインパターンスキル

デザインパターンドキュメントを作成・参照・レビューする際のドメイン知識。テンプレート、品質基準、カタログ管理規約を定義する。

## スコープ

- **提供するもの**: パターンのテンプレート、品質基準、カタログ管理規約
- **提供しないもの**: 作成手順（Agent が定義）、プロジェクト固有のパターン内容

## ファイル構成

```
docs/design-patterns/
├── index.md              # カタログ（全パターンの一覧と適用条件）
└── {concern}.md          # 個別パターン（1関心事 = 1ファイル）
```

### カタログ（index.md）の役割

- 全パターンの一覧と適用条件を記載
- Agentはカタログを参照し、タスクに必要なパターンのみを読み込む
- 新パターン作成時は必ずカタログも更新する

### 個別パターンの命名規則

```
{concern}.md
```

`concern` は技術的関心事を表す kebab-case の名前（例: `data-fetching.md`, `ui-components.md`, `state-management.md`）

## 読み込み戦略

デザインパターンは**選択的読み込み**で使用する。

```
実装タスク開始
  └─ design-patterns/index.md を参照
       └─ タスクに該当するパターンのみ読み込み
```

## 品質基準

### カタログ（index.md）のチェック項目

- [ ] 全パターンが一覧に含まれているか
- [ ] 各パターンの適用条件が明確か
- [ ] パターン間の関係（依存・競合）が記載されているか

### 個別パターンのチェック項目

- [ ] パターンの目的と適用場面が明確か
- [ ] 具体的なコード例が含まれているか
- [ ] アンチパターン（やってはいけないこと）が記載されているか
- [ ] 設定値の推奨値と根拠が示されているか
- [ ] 関連する他のパターンへの参照があるか
