# 用語集作成スキル

用語集を作成・レビューする際のドメイン知識。テンプレートと品質基準を定義する。

## スコープ

- **提供するもの**: 用語集のテンプレート、品質基準、分類体系
- **提供しないもの**: 作成手順（Agent が定義）、プロジェクト固有の用語

## 入出力

| 項目 | 内容 |
|---|---|
| 入力 | Step 1〜2 の全出力ドキュメント（PRD, 機能設計書, 画面仕様書, アーキテクチャ設計書, 開発ガイドライン, リポジトリ構造定義書） |
| 出力 | `docs/glossary.md` |

## 既存ドキュメントの優先順位

| 優先度 | 対象 | 説明 |
|---|---|---|
| 1（最優先） | `docs/glossary.md` | プロジェクト固有の用語定義。このスキルのガイドより優先 |
| 2（参考） | このスキル | 汎用テンプレート。既存用語集がない場合に使用 |

## テンプレート

用語集を作成する際は次のテンプレートを使用する: ./template.md

## 用語の分類体系

| カテゴリ | 内容 | 例 |
|---|---|---|
| ドメイン用語 | プロジェクト固有のビジネス概念・機能 | 「タスク」「ワークスペース」 |
| 技術用語 | 使用している技術・フレームワーク | 「TanStack Query」「Prisma」 |
| 略語・頭字語 | 略称とその正式名称 | 「PRD」「MVP」 |
| アーキテクチャ用語 | システム設計の概念 | 「レイヤードアーキテクチャ」「フィーチャーモジュール」 |
| ステータス・状態 | システム内の各種ステータス | 「未着手」「進行中」「完了」 |
| データモデル用語 | エンティティとその属性 | 「Userエンティティ」「Taskエンティティ」 |

## 品質基準

### 必須チェック項目

- [ ] 用語が上記の分類体系に沿って分類されているか
- [ ] 各用語に明確な定義があるか
- [ ] 具体的な使用例が含まれているか
- [ ] 関連用語が相互参照されているか
- [ ] ドメイン用語に英語表記が含まれているか
- [ ] 技術用語にプロジェクトでの用途とバージョンが記載されているか
- [ ] ステータス用語に状態遷移図が含まれているか
