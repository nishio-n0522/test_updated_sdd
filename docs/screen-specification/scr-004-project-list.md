## 概要

ホーム画面の「検査処理設定」から遷移するプロジェクト管理画面。検査フロー・AIモデルをまとめる単位である「プロジェクト」の一覧表示、新規作成、削除を行う。ユーザーはプロジェクトカードをダブルクリックしてフローエディタ（SCR-005）へ進む。MVP時点でのプロジェクト作成ダイアログはプロジェクト名の入力のみ。Phase 4以降でテンプレート選択UIがダイアログ内に追加される。

## 操作挙動仕様

### 操作: 新規プロジェクト作成ボタンをクリック

**トリガー**: ユーザーが「新規プロジェクト作成」ボタンをクリックする
**前提条件**: なし（常に操作可能）

**正常系の挙動**:

1. モーダルオーバーレイが表示され、プロジェクト作成ダイアログが画面中央に表示される
2. プロジェクト名入力フィールドに自動フォーカスが当たる
3. MVP時点では、プロジェクト名入力フィールドとテンプレート案内テキスト（「テンプレート選択は Phase 4 以降で利用可能」）が表示される
4. 「作成」ボタンは非活性状態（プロジェクト名が未入力のため）で表示される

**異常系の挙動**:

| 条件     | 挙動 |
| -------- | ---- |
| 該当なし | -    |

**画面遷移**: 遷移なし（モーダルダイアログ表示）
**状態変化**: プロジェクト一覧表示 → プロジェクト作成ダイアログ表示中

---

### 操作: プロジェクト名の入力

**トリガー**: ユーザーがプロジェクト名入力フィールドにテキストを入力する
**前提条件**: プロジェクト作成ダイアログが表示されている

**正常系の挙動**:

1. 入力した文字がリアルタイムでテキストフィールドに表示される
2. 1文字以上入力された時点で「作成」ボタンが有効状態（青色・強調表示）に変わる
3. 入力内容をすべて削除すると「作成」ボタンが再び非活性状態に戻る

**異常系の挙動**:

| 条件                        | 挙動                                                                               |
| --------------------------- | ---------------------------------------------------------------------------------- |
| 100文字を超えて入力しようとした場合 | 101文字目以降は入力を受け付けない。現在の文字数と上限をフィールド下部に表示する（例: 「100 / 100」） |

**画面遷移**: 遷移なし
**状態変化**: 「作成」ボタンの活性/非活性状態がリアルタイムで変化する

### 操作: プロジェクト作成ダイアログで「作成」をクリック

**トリガー**: ユーザーがプロジェクト作成ダイアログの「作成」ボタンをクリックする（またはEnterキーを押す）
**前提条件**: プロジェクト名が1文字以上入力されている（「作成」ボタンが有効状態）

**正常系の挙動**:

1. ダイアログが閉じる
2. 入力されたプロジェクト名でプロジェクトが作成される
3. MVP時点ではテンプレートなし（空のキャンバス）でプロジェクトが作成される
4. フローエディタ画面（SCR-005）へ遷移する。ヘッダーに作成したプロジェクト名が表示された空のキャンバスが表示される

**異常系の挙動**:

| 条件                                         | 挙動                                                                               |
| -------------------------------------------- | ---------------------------------------------------------------------------------- |
| プロジェクト名が未入力の状態でEnterキーを押した場合 | 「作成」ボタンが非活性のため何も起こらない                                      |
| 同名のプロジェクトが既に存在する場合         | 入力フィールド下部に「同じ名前のプロジェクトが既に存在します」とインラインエラーを表示する。別の名前の入力を促す。遷移は行わない |
| プロジェクトの作成に失敗した場合（ファイルI/Oエラー等） | ダイアログを維持し、エラーメッセージをダイアログ内に表示する。「再試行」でもう一度作成を試みられる |

**画面遷移**: SCR-005（フローエディタ画面）
**状態変化**: プロジェクト作成ダイアログ表示中 → SCR-005表示（新規プロジェクト・空キャンバス）

## ビジネスルール補足

- プロジェクトのデータ（フロー設定・AIモデル）はローカルにのみ保存される（クラウド連携を明示的に有効化するまで）
- プロジェクトは独立しており、異なるプロジェクトの設定が混在しない
- 同一の画像データセットを複数のプロジェクトから参照できる（Phase 1以降）
- テンプレート機能（FN-009）はPhase 4で有効化される。MVP時点ではプロジェクト作成ダイアログにテンプレート選択UIは表示せず、案内テキストのみ表示する
- テンプレート未選択またはMVP時点での作成は常に空のキャンバスで開始する
