## 概要

アプリ起動時に最初に表示される起点画面。上部にアプリロゴを配置したヘッダーバーを持ち、コンテンツ領域には3列×2行のグリッドレイアウトで各機能エリアへのナビゲーションボタンを配置する。左端にはサイドバー（グローバルナビゲーション）が閉状態で表示され、ホバー時に展開する。Post-MVP機能のボタンはMVP時点でもグレーアウト状態で表示し、将来フェーズで有効化される。

## 操作挙動仕様

### 操作: サイドバーのホバーによる展開・収納

**トリガー**: マウスカーソルがサイドバー領域に入る / 離れる
**前提条件**: ホーム画面（SCR-001）が表示されている

**正常系の挙動**:

1. マウスカーソルがサイドバー領域に入ると、サイドバーが展開しアイコン＋ラベルが表示される
2. 展開したサイドバーはコンテンツ領域に重なる形（オーバーレイ方式）で表示される。コンテンツのレイアウトは変化しない
3. マウスカーソルがサイドバー領域から離れると、サイドバーが閉状態に戻りアイコンのみの表示になる
4. 展開中は「ホーム」項目がアクティブ状態（ハイライト）で表示される

**異常系の挙動**:

| 条件         | 挙動             |
| ------------ | ---------------- |
| 該当なし     | -                |

**画面遷移**: 遷移なし
**状態変化**: サイドバー閉状態 ↔ サイドバー展開状態

---

### 操作: 有効な機能ボタンをクリック（検査処理設定）

**トリガー**: ユーザーが「検査処理設定」ボタン（グリッド内またはサイドバー展開時）をクリックする
**前提条件**: MVP機能のため常に有効状態

**正常系の挙動**:

1. ボタンのクリックフィードバック（押下エフェクト）を表示する
2. プロジェクト一覧画面（SCR-004）へ遷移する。サイドバーは開状態で表示され「検査処理設定」がアクティブ状態になる

**異常系の挙動**:

| 条件         | 挙動             |
| ------------ | ---------------- |
| 該当なし     | -                |

**画面遷移**: SCR-004（プロジェクト一覧画面）
**状態変化**: ホーム画面（SCR-001）→ プロジェクト一覧画面（SCR-004）

---

### 操作: 非活性ボタンをクリック（画像管理・学習管理・検査処理分析・検査稼働・クラウド）

**トリガー**: ユーザーがグレーアウト状態のボタン（グリッド内またはサイドバー展開時）をクリックする
**前提条件**: MVP時点ではPost-MVP機能ボタンがすべて非活性状態

**正常系の挙動**:

1. クリックされたボタンの近傍に「この機能は今後対応予定です」というツールチップが表示される
2. ツールチップは約2秒後に自動的に消える
3. 画面遷移は発生しない。ホーム画面を維持する

**異常系の挙動**:

| 条件         | 挙動             |
| ------------ | ---------------- |
| 該当なし     | -                |

**画面遷移**: 遷移なし（SCR-001のまま）
**状態変化**: 変化なし（ツールチップ表示のみ）

---

### 操作: グローバルナビの「ホーム」をクリック（他画面からの帰還時）

**トリガー**: 他の画面（SCR-004等）でサイドバーの「ホーム」をクリックする
**前提条件**: ユーザーが他の画面（ホーム以外）を表示している

**正常系の挙動**:

1. ホーム画面（SCR-001）へ遷移する
2. サイドバーが閉状態で表示され、グリッドレイアウトのコンテンツが表示される

**異常系の挙動**:

| 条件                                           | 挙動                                                                     |
| ---------------------------------------------- | ------------------------------------------------------------------------ |
| フローエディタ（SCR-005）で未保存の変更がある  | 保存確認ダイアログを表示する。「保存して移動」「破棄して移動」「キャンセル」の選択肢を提示する。保存または破棄を選択した場合のみホーム画面へ遷移する |

**画面遷移**: SCR-001（ホーム画面）
**状態変化**: 遷移元の画面 → SCR-001

---

### 操作: アプリ起動時の初期表示

**トリガー**: ユーザーがアプリを起動する（コールドスタート）
**前提条件**: アプリが起動していない状態

**正常系の挙動**:

1. スプラッシュ画面（ローディング表示）が表示される
2. 前回終了時の画面状態の復元を試みる
3. 前回状態がない場合、またはホーム画面で終了した場合は、ホーム画面（SCR-001）を表示する
4. 前回状態が別の画面（例: SCR-004）の場合は、その画面を復元して表示する（SCR-001はスキップされる）

**異常系の挙動**:

| 条件                             | 挙動                                                                                       |
| -------------------------------- | ------------------------------------------------------------------------------------------ |
| 前回クラッシュした場合           | リカバリーダイアログを表示し、「自動保存データから復元」または「新規起動（ホーム画面）」を選択できる |
| 自動保存データが破損している場合 | 復元に失敗した旨をトーストで通知し、ホーム画面（SCR-001）で新規起動する                     |

**画面遷移**: SCR-001（前回状態がない場合）または前回終了時の画面
**状態変化**: 起動前 → ホーム画面表示（または前回状態の復元）
