## 操作挙動仕様

### 操作: フローエディタ画面の初期表示

**トリガー**: SCR-004からプロジェクトを開く（ダブルクリックまたはプロジェクト作成後の遷移）
**前提条件**: SCR-004が表示されている

**正常系の挙動**:

1. ヘッダーにプロジェクト名・保存状態インジケーター・デバッグ実行ボタン・設定ボタンが表示される
2. サイドバーが開状態で表示され、「検査処理設定」がアクティブ状態になる
3. ノードパレットが表示され、全カテゴリが展開状態で表示される
4. 既存プロジェクトの場合: 保存済みフロー設定を読み込み、ノードとワイヤーをキャンバスに表示する。読み込み中はキャンバスにローディングインジケーターを表示する
5. 新規プロジェクトの場合: 空のキャンバスを表示し、中央に空状態ガイダンスメッセージを表示する

**異常系の挙動**:

| 条件                                       | 挙動                                                                                       |
| ------------------------------------------ | ------------------------------------------------------------------------------------------ |
| フロー設定ファイルが見つからない場合       | ウィンドウ内トーストで「フロー設定を読み込めませんでした」と通知し、空のキャンバスで表示する |
| フロー設定ファイルが破損している場合       | ウィンドウ内トーストで「フロー設定が破損しています。自動保存データから復元を試みます」と通知し、自動保存データからの復元を試みる。復元不可の場合は空のキャンバスで表示する |

**画面遷移**: 遷移なし（SCR-005表示）
**状態変化**: ローディング中 → データあり（または空状態）

---

### 操作: ノードをキャンバスに追加（ドラッグ）

**トリガー**: ユーザーがノードパレット内のノードカードをキャンバス上にドラッグ＆ドロップする
**前提条件**: ノードカードが有効状態（非活性でない）

**正常系の挙動**:

1. ドラッグ中はノードカードのゴーストプレビューがカーソルに追従する
2. キャンバス上でドロップすると、ドロップ位置にノードが配置される
3. 配置されたノードは選択状態（青色枠）になり、プロパティパネルが開いてデフォルト値が表示される
4. 自動保存タイマーがリセットされ、30秒以内に自動保存される
5. 保存状態インジケーターが「● 未保存」に変わる

**異常系の挙動**:

| 条件                                           | 挙動                                                                         |
| ---------------------------------------------- | ---------------------------------------------------------------------------- |
| キャンバス外にドロップした場合                 | ノードは配置されない。ドラッグキャンセルとして扱う                           |
| 非活性ノードカード（Phase 3〜）をドラッグした場合 | ドラッグを受け付けない。カーソルに禁止アイコンを表示し「Phase 3以降で利用可能」のツールチップを表示する |

**画面遷移**: 遷移なし
**状態変化**: キャンバスにノードが追加される。プロパティパネルが表示される

---

### 操作: ノードをキャンバスに追加（ダブルクリック）

**トリガー**: ユーザーがノードパレット内のノードカードをダブルクリックする
**前提条件**: ノードカードが有効状態（非活性でない）

**正常系の挙動**:

1. 現在表示されているキャンバスの中央にノードが配置される
2. 配置されたノードは選択状態（青色枠）になり、プロパティパネルが開いてデフォルト値が表示される
3. 自動保存タイマーがリセットされ、30秒以内に自動保存される
4. 保存状態インジケーターが「● 未保存」に変わる

**異常系の挙動**:

| 条件                                               | 挙動                                                                         |
| -------------------------------------------------- | ---------------------------------------------------------------------------- |
| 非活性ノードカード（Phase 3〜）をダブルクリックした場合 | ノードは配置されない。「Phase 3以降で利用可能です」のツールチップを表示する |

**画面遷移**: 遷移なし
**状態変化**: キャンバスにノードが追加される

---

### 操作: ノードを選択（クリック）

**トリガー**: ユーザーがキャンバス上のノードをクリックする
**前提条件**: キャンバスにノードが1件以上配置されている

**正常系の挙動**:

1. クリックされたノードが選択状態（青色強調枠）になる
2. 以前選択されていたノードの選択が解除される
3. プロパティパネルが表示（または更新）され、選択ノードのパラメータが表示される
4. 操作はUIの応答時間200ms以内に完了する

**異常系の挙動**:

| 条件                | 挙動 |
| ------------------- | ---- |
| 該当なし            | -    |

**画面遷移**: 遷移なし
**状態変化**: 未選択 → ノード選択状態（プロパティパネル表示）

---

### 操作: ノードの選択解除（キャンバス空白クリック）

**トリガー**: ユーザーがキャンバスの何もない領域をクリックする
**前提条件**: ノードが選択状態になっている

**正常系の挙動**:

1. すべてのノードの選択が解除される
2. プロパティパネルが非表示になり、キャンバスが右端まで広がる

**異常系の挙動**:

| 条件     | 挙動 |
| -------- | ---- |
| 該当なし | -    |

**画面遷移**: 遷移なし
**状態変化**: ノード選択状態 → 未選択（プロパティパネル非表示）

---

### 操作: ノードを移動（ドラッグ）

**トリガー**: ユーザーがキャンバス上のノードをドラッグする
**前提条件**: キャンバスにノードが1件以上配置されている

**正常系の挙動**:

1. ドラッグ中はノードがカーソルに追従してリアルタイムで移動する
2. ノードに接続されたワイヤーも追従してリアルタイムで更新される
3. ドロップ位置でノードが確定する
4. 自動保存タイマーがリセットされ、30秒以内に自動保存される
5. 保存状態インジケーターが「● 未保存」に変わる
6. 操作はUIの応答時間200ms以内に完了する

**異常系の挙動**:

| 条件     | 挙動 |
| -------- | ---- |
| 該当なし | -    |

**画面遷移**: 遷移なし
**状態変化**: ノード位置が更新される

---

### 操作: ノードを削除（Deleteキー）

**トリガー**: ユーザーがノードを選択状態にしてDeleteキーを押す
**前提条件**: ノードが選択状態になっている

**正常系の挙動**:

1. 選択されたノードがキャンバスから削除される
2. 削除されたノードに接続されたすべてのワイヤーも同時に削除される
3. プロパティパネルが非表示になる
4. 自動保存タイマーがリセットされ、30秒以内に自動保存される
5. 保存状態インジケーターが「● 未保存」に変わる

**異常系の挙動**:

| 条件     | 挙動 |
| -------- | ---- |
| 該当なし | -    |

**画面遷移**: 遷移なし
**状態変化**: ノードがキャンバスから削除される

---

### 操作: ノードを削除（コンテキストメニュー）

**トリガー**: ユーザーがノードを右クリックし、コンテキストメニューの「削除」を選択する
**前提条件**: キャンバスにノードが1件以上配置されている

**正常系の挙動**:

1. 右クリックしたノードの近傍にコンテキストメニューが表示される（削除・複製・ノード情報確認）
2. 「削除」を選択すると、ノードとそれに接続されたワイヤーが削除される
3. プロパティパネルが非表示になる
4. 保存状態インジケーターが「● 未保存」に変わる

**異常系の挙動**:

| 条件     | 挙動 |
| -------- | ---- |
| 該当なし | -    |

**画面遷移**: 遷移なし
**状態変化**: ノードがキャンバスから削除される

### 操作: ノード間の接続（ワイヤー作成）

**トリガー**: ユーザーがノードの出力ポートからドラッグし、別ノードの入力ポートにドロップする
**前提条件**: キャンバスに2件以上のノードが配置されている

**正常系の挙動**:

1. 出力ポートからドラッグを開始すると、カーソルに追従するワイヤーのプレビューが表示される
2. 接続可能な入力ポートにカーソルが近づくと、ポートがハイライト表示される
3. 入力ポートにドロップするとワイヤーが確定し、ノードが接続される
4. 自動保存タイマーがリセットされ、30秒以内に自動保存される
5. 保存状態インジケーターが「● 未保存」に変わる
6. 操作はUIの応答時間200ms以内に完了する

**異常系の挙動**:

| 条件                                                 | 挙動                                                                                                   |
| ---------------------------------------------------- | ------------------------------------------------------------------------------------------------------ |
| 型が一致しないポートにドロップした場合               | 接続を拒否する。ポートを赤色でフィードバックし「型が一致しません（[出力型] → [入力型]）」のツールチップを表示する |
| 入力ポート以外の場所にドロップした場合               | ワイヤーは作成されない。ドラッグキャンセルとして扱う                                                   |
| すでに接続されている入力ポートに再接続しようとした場合 | 既存のワイヤーを切断し、新しいワイヤーで接続する（入力ポートは1接続のみ）                              |
| 同一ノードの出力ポートから同じノードの入力ポートへの接続 | 循環接続として拒否する。「同一ノードへの接続はできません」のツールチップを表示する                  |

**画面遷移**: 遷移なし
**状態変化**: ノード間にワイヤーが追加される

### 操作: ワイヤーの削除

**トリガー**: ユーザーがワイヤーをクリックして選択後、Deleteキーを押す
**前提条件**: キャンバスにワイヤーが1件以上存在する

**正常系の挙動**:

1. ワイヤーをクリックするとワイヤーが選択状態（太く強調表示）になる
2. Deleteキーを押すとワイヤーが削除される
3. 保存状態インジケーターが「● 未保存」に変わる

**異常系の挙動**:

| 条件     | 挙動 |
| -------- | ---- |
| 該当なし | -    |

**画面遷移**: 遷移なし
**状態変化**: ワイヤーが削除される

### 操作: パラメータを変更（プロパティパネル）

**トリガー**: ユーザーがプロパティパネル内のパラメータ入力フィールド・ドロップダウンを変更する
**前提条件**: ノードが選択状態でプロパティパネルが表示されている

**正常系の挙動**:

1. 入力フィールドへのテキスト入力、ドロップダウンの選択変更がリアルタイムに反映される
2. 変更後すぐに（200ms以内）ノードカードの表示が更新される（表示上の変化がある場合）
3. 自動保存タイマーがリセットされ、30秒以内に自動保存される
4. 保存状態インジケーターが「● 未保存」に変わる

**異常系の挙動**:

| 条件                                             | 挙動                                                                                       |
| ------------------------------------------------ | ------------------------------------------------------------------------------------------ |
| 入力値がパラメータの制約範囲を外れている場合     | 入力フィールド下部にインラインエラーメッセージを表示する（例: 「カーネルサイズは奇数を入力してください」）。デバッグ実行ボタンを非活性にする |
| 入力値が空になった場合                           | フィールドをエラー状態（赤枠）で表示し、「必須パラメータです」のメッセージを表示する    |

**画面遷移**: 遷移なし
**状態変化**: ノードのパラメータが更新される

### 操作: プレビュー更新ボタンをクリック

**トリガー**: ユーザーがプロパティパネル内の「プレビュー更新」ボタンをクリックする
**前提条件**: ノードが選択状態でプロパティパネルが表示されている。フローに画像入力ノードが存在し、入力画像が設定されている

**正常系の挙動**:

1. 選択ノードから入力ノードまで遡って中間実行を行う
2. 実行中はノードカードにスピナーを表示する
3. 実行完了後、ノードカード下部に処理結果のサムネイル画像を表示する

**異常系の挙動**:

| 条件                                 | 挙動                                                                         |
| ------------------------------------ | ---------------------------------------------------------------------------- |
| 入力画像が未設定の場合               | 「画像入力ノードに画像を設定してください」のウィンドウ内トーストを表示する   |
| パラメータに入力エラーがある場合     | 「パラメータのエラーを修正してからプレビューを実行してください」と表示する   |
| 実行中にエラーが発生した場合         | ノードカードにエラーバッジを表示し、エラー内容をツールチップで確認できるようにする |

**画面遷移**: 遷移なし
**状態変化**: ノードカードに中間出力サムネイルが表示される

### 操作: デバッグ実行ボタンをクリック

**トリガー**: ユーザーがヘッダーの「▷ 実行」ボタンをクリックする
**前提条件**: キャンバスにノードが1件以上配置されており、パラメータエラーが存在しない

**正常系の挙動**:

1. ヘッダーのボタンが「■ 停止」に切り替わる
2. フローの先頭（入力ノード）から順番に各ノードが実行される
3. 実行中のノードにスピナーバッジが表示される
4. 実行完了したノードにチェックバッジと中間出力サムネイルが表示される
5. すべてのノードの実行が完了すると、ボタンが「▷ 実行」に戻る
6. 結果出力ノードに最終判定結果が表示される

**異常系の挙動**:

| 条件                                       | 挙動                                                                                                   |
| ------------------------------------------ | ------------------------------------------------------------------------------------------------------ |
| ノードが0件の場合                          | ボタンが非活性のため何も起こらない                                                                     |
| パラメータにエラーがある場合               | ウィンドウ内トーストで「パラメータのエラーを修正してから実行してください」と通知し、実行しない        |
| 入力ノードに画像が設定されていない場合     | ファイル選択ダイアログを表示してテスト用画像を選択させる。キャンセル時は実行しない                    |
| フロー実行中にノードでエラーが発生した場合 | エラーが発生したノードにエラーバッジを表示し、ノードカードにエラー詳細テキストを表示する。後続ノードの実行を停止する。ボタンが「▷ 実行」に戻る |
| 「■ 停止」ボタンをクリックした場合         | 現在実行中のノードの処理を中断し、実行状態をリセットする。ボタンが「▷ 実行」に戻る                    |

**画面遷移**: 遷移なし
**状態変化**: 通常 → デバッグ実行中 → 実行完了（またはエラー）

### 操作: 実行完了ノードの中間出力を確認

**トリガー**: ユーザーがデバッグ実行完了後のノードに表示されているサムネイルをクリックする
**前提条件**: デバッグ実行が完了しており、ノードに中間出力サムネイルが表示されている

**正常系の挙動**:

1. モーダルダイアログで中間出力画像が拡大表示される
2. 元の入力画像と処理後の画像を並べて確認できる（ノード種別により異なる）
3. ダイアログを閉じるとキャンバスに戻る

**異常系の挙動**:

| 条件     | 挙動 |
| -------- | ---- |
| 該当なし | -    |

**画面遷移**: 遷移なし（モーダルダイアログ表示）
**状態変化**: 中間出力プレビューダイアログ表示

### 操作: キャンバスのズーム（マウスホイール）

**トリガー**: ユーザーがキャンバス上でマウスホイールを回転させる、またはズームイン/アウトボタンをクリックする
**前提条件**: キャンバスが表示されている

**正常系の挙動**:

1. マウスカーソル位置を中心にズームイン/アウトが行われる（ボタン操作時はキャンバス中央基準）
2. ノードとワイヤーがリアルタイムで拡大・縮小される
3. 操作はUIの応答時間200ms以内に完了する

**異常系の挙動**:

| 条件                     | 挙動                                                             |
| ------------------------ | ---------------------------------------------------------------- |
| 最大ズーム率（200%）に達した場合 | それ以上のズームインを受け付けない。ズームインボタンが非活性になる |
| 最小ズーム率（10%）に達した場合  | それ以上のズームアウトを受け付けない。ズームアウトボタンが非活性になる |

**画面遷移**: 遷移なし
**状態変化**: キャンバスのズーム倍率が変化する

**トリガー**: ユーザーがキャンバス右下の「[⊞] 全体表示」ボタンをクリックする

| ノードが1件で画面内に収まっている場合 | ズーム・パンの変更は最小限または行わない            |

**画面遷移**: 遷移なし

### 操作: 「プロジェクト一覧」へ戻る（未保存変更がある場合）

**トリガー**: ユーザーがサイドバーの「検査処理設定」ナビ項目をクリックする（未保存変更がある状態）
**前提条件**: フローに未保存の変更がある（「● 未保存」を表示）

**正常系の挙動**:

1. 保存確認ダイアログが表示される。「保存して移動」「破棄して移動」「キャンセル」の選択肢を提示する
2. 「保存して移動」を選択した場合: フローを保存してSCR-004へ遷移する
3. 「破棄して移動」を選択した場合: 未保存の変更を破棄してSCR-004へ遷移する
4. 「キャンセル」を選択した場合: ダイアログを閉じてSCR-005のままにする

**異常系の挙動**:

| 条件                                     | 挙動                                                                                     |
| ---------------------------------------- | ---------------------------------------------------------------------------------------- |
| 「保存して移動」選択時に保存が失敗した場合 | ダイアログを維持し、「保存に失敗しました。再試行するか、破棄して移動してください」を表示する |

**画面遷移**: SCR-004（「保存して移動」または「破棄して移動」選択時）/ 遷移なし（「キャンセル」選択時）
**状態変化**: 保存確認ダイアログ表示 → SCR-004（またはSCR-005維持）

## ビジネスルール補足

- フローはプロジェクトに紐づく。1つのプロジェクトに1つのフローが存在する
- フロー設定はノードの追加・削除・接続・パラメータ変更が行われるたびに、30秒以内に自動保存される。手動保存でも即時保存できる
- 50ノード以上で構成されるフローを問題なく設計・実行できるよう、キャンバスのズーム・パン機能を提供する
- ノードの接続は出力ポートと入力ポートのデータ型（画像・数値・判定結果など）の互換性に基づいて制限する。型が一致しない接続は拒否する
- フロー編集操作（ノード追加・接続・パラメータ変更）に対するUIの応答時間は200ms以内とする
- AI推論ノード（Phase 3〜）はMVP時点でノードパレットに表示するが非活性状態（破線枠・グレーアウト）とし、Phase 3で有効化する
- デバッグ実行（試験実行）はフロー評価機能（SCR-007）とは概念的に異なり、エディタ上での中間出力確認を目的とする。確定的な品質保証には使用しない
- AI推論ノードの設定から「学習を開始」をクリックすると学習管理画面（SCR-006）へ遷移する（Phase 3〜）。学習完了後は「フローエディタへ戻る」でSCR-005に戻ることができる
